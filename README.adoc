= JAP Dashboard

A dashboard for managing and monitoring JAP (Java Application Platform) services. The dashboard provides visibility into projects, artifacts, deployments, and integrations across multiple platforms.

== Overview

JAP Dashboard is a full-stack application that aggregates data from multiple services and presents it through an intuitive web interface. It enables teams to monitor project status, artifact reports, and deployments via integrations with GitLab, Harbor, Nomad, and LogScale.

=== Key Features

* *Project Management*: View and manage projects with detailed information
* *Artifact Reporting*: Track and analyze artifact reports across projects
* *Multi-Platform Integration*:
** GitLab: Project and activity information
** Harbor: Container registry and artifact management
** Nomad: Job orchestration and deployment status
** LogScale: Log aggregation and analysis
* *Real-time Updates*: React-based frontend with reactive data fetching

== Getting Started

=== Prerequisites

* *Java 21*: Required for backend
* *Gradle*: Build tool (wrapper included)
* *Node.js 18+*: Required for frontend
* *pnpm*: Package manager for frontend

== Configuration

=== Application Properties

Configuration is managed through YAML files in `src/main/resources/`:

* *application.yml*: Default settings
* *application-dev.yml*: Local development overrides

=== Backend Setup

== Variables
Following environment variables are necessary for fetching data from external services. For local development, it is possible to use personal credentials, but for production, it is recommended to create a service account.

* GITLAB_ACCESS_TOKEN: Personal access token for GitLab API - can be generated in GitLab under User Settings > Access Tokens

* NOMAD_SECRET_ID: Secret ID for Nomad API authentication - can be created in Nomad under Access Control > Secrets

* AD_USER: Active Directory username for authentication - must have appropriate permissions to access required resources
* AD_PASSWORD: Active Directory password for authentication

. Build and run:
+
[source,bash]
----
# Build the project
./gradlew build

# Run the application
./gradlew run
----
+
The backend will be available at `http://localhost:8080`

. View API Documentation:
+
* Swagger UI: `http://localhost:8080/swagger-ui.html`
* OpenAPI spec: `http://localhost:8080/openapi.yaml`

=== Frontend Setup

. Navigate to UI directory:
+
[source,bash]
----
cd ui
----

. Install dependencies:
+
[source,bash]
----
pnpm install
----

. Development server:
+
[source,bash]
----
pnpm dev
----
+
Frontend will be available at `http://localhost:5173`
